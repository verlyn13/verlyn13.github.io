Flux Pro Shop

Image Generation Control Plane

Working prototype · Pre-cutover

Overview

An image-generation control plane built on HuggingFace Diffusers, optimized for Apple Silicon (MPS). It compiles a single generation intent across a registry of providers, manages local GPU and remote API runners, and captures artifact lineage in content-addressed storage. This is a self-directed working prototype, pre-cutover, not a shipped product.

By designing a unified Compile/Execute Intermediate Representation (IR), this architecture abstracts the generation paths for complex ML models (Flux, SDXL/Pony, and GGUF).

Technical Architecture

Core Engine & IR

  • Python 3.12+ type-safe core
  • Unified IR: WorkflowSpec → Compiler → ExecutionPlan → Runner → Result
  • Backend abstraction (DiffusersBackend, SDXLBackend, ComfyUIBackend, ApiBackend)

Data & Lineage

  • Content-addressed storage for immutable generation results
  • Automated lineage tracking (source_run, derivation)
  • JSON-based metadata-driven reproducibility

Why This Matters

Artifact lineage

Content-addressed storage and lineage tracking make generation results traceable and reproducible.

Deterministic execution

A unified compile/execute IR (WorkflowSpec → Compiler → ExecutionPlan → Runner) gives repeatable generation across backends.

Hardware abstraction

Dynamic routing between Apple Silicon (MPS) and remote API execution environments.

Engineering discipline

A typed Python core, backend abstraction, and content-addressed artifacts bring software-engineering rigor to an experimental ML pipeline.

Technical Depth

This project demonstrates:

  • Compile/execute IR design (WorkflowSpec → Compiler → ExecutionPlan → Runner)
  • Backend abstraction across Diffusers, SDXL/Pony, ComfyUI, and API runners
  • Content-addressed storage and automated artifact lineage
  • Local GPU (Apple Silicon / MPS) and remote API execution routing
  • A type-safe Python 3.12+ core

Development Context

A self-directed working prototype, pre-cutover: the successor control plane for image generation, consolidating multiple providers behind one compile/execute IR with reproducible, content-addressed artifacts. Not presented as a shipped product.
