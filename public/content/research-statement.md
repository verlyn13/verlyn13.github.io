# Research statement

**Jeffrey V. Johnson, Ph.D.** | Building infrastructure for model evaluation and
agent systems

I build the foundations that let agents contribute reliable,
specification-conforming work: typed contracts, bounded authority, reproducible
tool and runtime environments, deterministic gates, and evidence-preserving
handbacks. I test those foundations by using them to build complex applications,
data pipelines, evaluation systems, and infrastructure.

I am a tenured mathematics professor by background, trained in commutative
Banach algebras, and a self-directed systems researcher by current practice. I
have built a deliberate research-engineering program spanning specification
design, agent authority, workstation and runtime configuration, policy-as-code,
evaluation, infrastructure, data systems, and native applications. Agent tools
are instruments inside this program, not the source of quality.

## Foundation for reliable agent development

The **Host Capability Substrate** is the public implementation core. It treats
host facts, capabilities, operations, execution contexts, policy decisions,
provenance-typed evidence, approvals, leases, audits, and scoped authorization as
typed objects rather than implicit shell access. The point is not a better prompt.
The point is a system that can determine what work is admissible, what evidence
is required, and when it must refuse or return control to a human.

The anonymized **governance and agent-control architecture** applies that method
across people, agents, repositories, credentials, infrastructure, and runtime
systems. Its design separates decision rights, bounded mandates, capability and
credential reach, enforcement, admission, and repository-local authority.
Value-blind handbacks return privileged or judgment-heavy work to a human while
preserving scope, evidence, validation, unresolved risk, and the next responsible
actor.

Execution environment is part of the contract. Managed human and agent shell
modes, tool and runtime resolution, environment activation, and repository-local
rules make it possible for different coding agents and editors to work under the
same specification. Selected workstation, policy-as-code, branch-governance,
promotion, Zero Trust, and drift controls operate today. Runtime admission
remains source-tested rather than a live request-path gate, and the case study
makes no enterprise-adoption claim.

## Independent specification and verification

The public **Agentic Architecture Audit** checks whether agent-built systems
conform to their own specifications. Its two-stage, evidence-first `audit-spec`
separates discovery from judgment, defines structured output schemas, and uses
deterministic drift checks to keep derived documents bound to specification
bytes. Findings become repeatable fitness functions rather than remaining prose
recommendations.

This is a recurring principle in my work: deterministic checks should decide
what they can, probabilistic tools should remain bounded by evidence, and human
judgment should stay visible where semantics or authority cannot be reduced to a
gate.

## Proving grounds

**Budget Triage** tests the development substrate against a consequential private
financial application. Nineteen executable specification contracts,
machine-readable agent authority boundaries, fail-closed pre-write hooks, and a
34-gate validation chain govern work across tools. The application itself
preserves extraction provenance, exact monetary semantics, tenant isolation,
append-only decisions, and human acceptance of uncertain data. Its 6 passing / 9
pending synthetic known-answer result remains incomplete and uncertified. No
production, financial-outcome, live-provider-performance, or source-through-tax
claim is made.

**ScopeCam** tests the method across Android, Kotlin, C++, JNI/NDK, USB hardware,
concurrency, release boundaries, and physical-device evidence. Its operating
contract keeps source validation distinct from bench observation and requires
explicit PASS or FAIL handbacks. A signed client alpha was delivered, but the
private application is not broadly released and makes no arbitrary-device or
production-readiness claim.

A private **data-cleaning pipeline** applies typed records, validators,
checkpointed transformations, lineage fingerprints, and privacy-gated releases
to sensitive source material. The data and source remain private; the relevant
claim is the architecture and control method, not publication or model quality.

The **Agentic-Coding Evaluation Lab** is a specialized evaluation proving ground.
It preserves the inputs to a model-change decision, checks the evidence chain,
and uses a separate verifier to recompute a result from frozen bytes. Its current
synthetic example returns `NOT_EVALUABLE` because the scientific prerequisites
are incomplete. That demonstrates fail-closed reporting, not real-model
performance or a promotion recommendation.

## Mathematical habit

My doctoral work studied maps between algebras of functions and the conditions
that force their structure. The habit from that work is to define the objects,
state the constraints, and understand what follows. That habit transfers
directly to agent systems. An agent is an actor inside a structure of tools,
permissions, state, environments, and evidence. The design problem is to make
that structure explicit enough that its work can be checked.
