# ADR-0008: A colophon page presenting the governed publishing pipeline

- Status: Accepted · Date: 2026-06-16

## Context
The portfolio's project feed (`public/data/projects.json`, deployed at `/data/projects.json`) is
generated from a controlled inventory and delivered through a governed pipeline: a scheduled job
regenerates the feed from a pinned source revision, a no-op diff gate suppresses unchanged refreshes,
and a draft change is opened for the operator to merge. `scripts/check-source-provenance.mjs` enforces
the guarantees in CI — a provenance + leak gate, and a dirty source (`kbDirty: true`) is blocked from
deploying. This governance is a credibility signal, but it had **no public surface** — a visitor could
not see that the project set is machine-generated, provenance-checked, and approval-gated.

Presenting it touches several ask-first areas (a new page, the site map / footer, and a new component
pattern, per `AGENTS.md`), so it is recorded here.

## Decision
- Add a new page **`colophon.html`** ("how this portfolio is published"), wired as the Vite input
  `colophon`. It follows the standard subpage skeleton (`header → back-nav → main → footer`) at the
  prose `--measure`, with one `<h1>` and continuous `<h2>` sections. It is discoverable via a site-wide
  **footer colophon link** (`… · Colophon`) and is intentionally **not** added to the top nav.
- Add two token-only component clusters in `assets/jeffrey.css` (no new tokens):
  - **`.pipeline`** — a three-lane swimlane (source of truth → automation → public site). Steps connect
    via a CSS `::before` arrow; the single **`.pipeline-step--gate`** (human merge) carries the page's
    one `--accent` emphasis, and **`.pipeline-step--live`** marks the live endpoint with an `--accent-2`
    border + `--green-bg` tint and `--ink` text (kept at AA, unlike the lower-contrast
    `.project-status.production` pill it visually echoes).
  - **`.evidence-grid` / `.evidence-card`** (+ a **`.guarantee-list`** checklist) — compact, factual
    blocks about the feed.
- **Descriptive, not feed-coupled.** The page states the system's *stable* properties and links the live
  `/data/projects.json` for *current* values, rather than mirroring volatile per-refresh data (source
  SHAs, project counts, `asOf` dates). It therefore carries no `<!-- feed: -->` comment and stays out of
  the per-refresh churn.
- **Honesty + safety.** Freshness is framed as a "scheduled cadence with content-change detection,"
  never "real-time." The page names no secrets, tokens, or internal branch names, and never names the
  feed-suppressed project guarded by the checker's `forbidStrings` leak gate.

## Consequences
- Visitors get a public, checkable account of how the portfolio is published, with the human approval
  gate made explicit (the merge is the gate).
- Two components enter the catalog (recorded in `design/DESIGN_SPEC.md` + `docs/design-system.md`).
  `--accent` stays reserved (the human-merge gate is the page's single accented element).
- Because the page is authored, not regenerated by the feed job, its prose could drift from the feed;
  mitigated by stating only stable properties and linking the live artifact for current values.
- No change to tokens, theming, the single deployed stylesheet, the feed pipeline, or dependencies.
